# 智能锂离子电池储能管理系统

本项目实现了一个**用于微电网和可再生能源场景下锂离子电池储能系统 (LBSS) 监控与控制的智能 IoT 系统**。  
该项目是比萨大学 *物联网* 课程的学术项目。

系统结合了**边缘计算、机器学习和预测控制**，以延长电池寿命、确保安全，并优化电池、可再生能源与主电网之间的能量流动。

---

## 🚀 核心功能

- **边缘级电池安全与健康监控**
- **基于机器学习的实时健康状态 (SoH) 估算**
- **基于模型预测控制 (MPC) 的预测性能量调度**
- **多层 IoT 架构（边缘层、聚合层、云端层）**
- **基于 CoAP over 6LoWPAN 的低功耗通信**
- **通过 MQTT 实现异步告警与监控**
- **通过 REST API 进行远程监控与控制**
- **故障隔离与安全优先设计**

## 🏗 系统架构

架构分为三个逻辑层：

### 🔹 边缘层
- **BatteryController（电池控制器）**
  - 运行于 nRF52840 设备
  - 监测电压、电流、温度、荷电状态 (SoC)
  - 执行设备端 ML 模型估算电池健康状态 (SoH)
  - 执行安全约束并控制功率继电器
  - 保证对关键故障的毫秒级响应

### 🔹 聚合层
- **uGridController（微电网控制器）**
  - 连接混合逆变器及多个 BatteryController
  - 使用 ML 预测负载和可再生能源发电
  - 通过 MPC 优化充放电决策
  - 作为电池遥测数据的代理
- **RPL Border Router（RPL 边界路由器）**
  - 将低功耗无线传感器网络桥接到 IPv6 互联网

### 🔹 云端层
- **RCA（远程控制应用）**
  - REST API（Flask）用于监控与控制
  - 通过 MySQL 进行数据持久化
  - 通过 MQTT 分发告警信息
- **CA（客户端应用）**
  - 基于 CLI 的实时监控界面
  - 订阅 MQTT 告警
  - 支持手动控制和 MPC 参数调节

## 📐 分层架构详解

### 为什么必须分层？

| 维度 | 边缘层 (BatteryController) | 聚合层 (uGridController / Border Router) |
|------|---------------------------|----------------------------------------|
| **管理对象** | 单个电池 | 多个电池 + 逆变器 + 负载 |
| **决策类型** | 实时安全保护（毫秒级） | 全局优化调度（秒级） |
| **计算资源** | 极度受限（KB 级内存，MCU） | 相对充裕（MB/GB 级，SBC 或 x86） |
| **网络角色** | 叶子节点，通过 6LoWPAN 通信 | 父节点/网关，桥接低功耗网络与 IPv6 |
| **ML 任务** | 轻量推理（SoH 估算，C 头文件） | 较重推理（功率预测）+ MPC 优化 |

### 类比理解

- **BatteryController** = 汽车的 ABS 系统，必须在毫秒内独立做出安全决策，不能等上级批准
- **uGridController** = 交通调度中心，看全局路况，协调多辆车的路线
- **Border Router** = 高速公路收费站，负责不同网络之间的协议转换

### 分层的核心价值

1. **故障隔离**：单个电池出问题时，边缘层能立即隔离，不依赖上级
2. **带宽限制**：6LoWPAN 带宽极低（~250kbps），不能把所有原始数据都传到云端
3. **延迟要求**：安全保护必须本地毫秒级响应，等云端往返早就出事了

## 🧠 机器学习与控制

### 电池健康估算
- ML 模型**直接在电池控制器上运行**
- 根据近期遥测数据估算健康状态 (SoH)
- 支持主动维护和故障隔离

### 预测性能量调度
- 负载和光伏发电在本地进行预测
- MPC 优化目标：
  - 电网能量交换成本
  - 电池衰减
  - 荷电状态 (SoC) 稳定性
- 使用**投影梯度下降法**求解优化问题，适用于嵌入式设备

## 🌐 通信协议栈

- **CoAP** 用于轻量级设备通信
- **6LoWPAN + IEEE 802.15.4** 用于低功耗组网
- **MQTT** 用于异步告警
- **HTTP/REST** 用于云端与客户端交互
- **CBOR 与 JSON** 用于高效载荷编码

### 为什么用 CoAP 而不是 HTTP？

CoAP 常被理解为"给低功耗设备用的 HTTP"，但二者有本质差异：

| 维度 | HTTP | CoAP |
|------|------|------|
| **传输层** | TCP（可靠但重） | UDP（轻量，自己实现可靠性） |
| **头部大小** | 几十~几百字节 | 最小 4 字节（二进制编码） |
| **适用网络** | WiFi/以太网 | 6LoWPAN（MTU 仅 127 字节） |
| **服务器推送** | 需要 WebSocket/SSE | 原生支持 OBSERVE 订阅 |
| **分块传输** | 需要 HTTP/2+ | 原生支持 Block-wise 分块 |

**类比**：HTTP = 卡车运货（功能全，需要宽阔公路）；CoAP = 摩托车送货（精简，能在小巷穿行）。

**为什么 Smart-LBSS 必须用 CoAP？**
电池控制器运行在 nRF52840 上，RAM 仅 256KB，网络是 6LoWPAN（带宽 ~250kbps）。HTTP 的 TCP 三次握手和冗长头部开销在这种环境下根本无法承受。

## 🔋 实际应用场景

本项目面向**工业/电网级储能场景**，而非消费电子产品。通过 MPC 优化充放电策略、避免过充过放、控制充放电速率，可将电池组寿命从 5-8 年延长至 10-15 年，具有显著的经济价值。

### 发电侧
- **风电场配套储能电池** — 平抑风电波动，减少频繁充放电造成的衰减
- **光伏电站储能电池** — 白天储电、夜间放电，优化充放电节奏

### 电网侧
- **独立储能电站** — 大型锂电池阵列，用于调峰调频，MPC 可降低循环损耗
- **变电站储能** — 延缓电网升级投资，减少电池深度充放电次数

### 用户侧
- **工商业储能柜** — 工厂/商场的削峰填谷，降低电费，同时延长电池组寿命
- **户用储能系统** — 家庭太阳能 + 储能墙（类似 Tesla Powerwall）
- **微电网储能** — 海岛、偏远地区的离网/并网微电网
